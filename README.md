# MTB Galaxy

MTB Galaxy - демо-приложение для геймифицированной банковской лояльности. Пользователь прокачивает три планеты, выполняет квесты, получает награды, приглашает друзей и играет в мини-игры. Админская часть показывает KPI, симулирует банковские события и подсвечивает рискованные операции.

## Что внутри

- `apps/web` - клиентское приложение на React, Vite, TanStack Query, Zustand и Framer Motion.
- `apps/api` - FastAPI API с SQLite по умолчанию, SQLAlchemy-моделями, движком наград, квестами и риск-флагами.
- `packages/contracts` - общие TypeScript-типы и метаданные планет, синхронизированные с OpenAPI.
- `docker-compose.yml` - опциональные PostgreSQL и Redis для более боевого стенда.

## Основные сценарии

- Галактика: обзор профиля, энергии, планет, бустеров, квестов, наград и лидерборда.
- Квесты: прогресс по условиям, получение награды после выполнения.
- Награды: история начислений, контейнеры со звездной пылью, сводка мини-игр.
- Социальное кольцо: отправка приглашения и лидерборд.
- Змейка Халва: аркадная мини-игра для планеты покупок.
- Реактор щита: игра на точность для кредитной дисциплины.
- Сигнальный ринг: игра на память и реакцию для реферальной механики.
- Админка: KPI-витрина, симулятор событий, журнал риск-флагов и ожидающих наград.

## Требования

- Node.js 20 или новее.
- Python 3.12 или новее.
- Windows PowerShell, Git Bash или другая оболочка с доступом к `npm` и Python.

## Быстрый старт

Установите JavaScript-зависимости:

```bash
npm install
```

Создайте Python-окружение и установите API:

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -e apps/api[dev]
```

Сгенерируйте контракты из OpenAPI:

```bash
npm run contracts:generate
```

При необходимости скопируйте примеры переменных окружения:

```bash
copy apps\api\.env.example apps\api\.env
copy apps\web\.env.example apps\web\.env
```

Запустите API:

```bash
npm run dev:api
```

Запустите фронтенд:

```bash
npm run dev:web
```

Или поднимите весь демо-стенд одной командой:

```bash
npm run dev
```

После запуска откройте `http://localhost:5173`.

## Переменные окружения

API:

- `DATABASE_URL` - строка подключения к базе. По умолчанию используется локальная SQLite-база.
- `CORS_ORIGINS` - список разрешенных origin для фронтенда.
- `SSE_INTERVAL_SECONDS` - интервал обновления админского SSE-потока.

Фронтенд:

- `VITE_API_URL` - адрес API. Для локальной разработки обычно `http://localhost:8000`.
- `VITE_FEATURE_HALVA_SNAKE` - включает или выключает мини-игру «Змейка Халва».

## API

Все ручки доступны от корня API, например `http://localhost:8000/galaxy/profile`.

| Метод | Путь | Назначение |
| --- | --- | --- |
| `POST` | `/auth/demo-login` | Создает или обновляет демо-сессию пользователя. |
| `GET` | `/galaxy/profile` | Возвращает профиль галактики, планеты, бустеры, квесты и рассрочку. |
| `POST` | `/events/ingest` | Принимает банковское событие и отправляет его в движок обработки. |
| `GET` | `/quests` | Возвращает список квестов пользователя. |
| `POST` | `/quests/{quest_id}/claim` | Выдает награду за выполненный квест. |
| `GET` | `/rewards/ledger` | Возвращает журнал начислений и наград. |
| `POST` | `/games/runs` | Сохраняет результат мини-игры и начисляет звездную пыль. |
| `GET` | `/games/summary` | Возвращает агрегированную статистику по мини-играм. |
| `POST` | `/referrals/invite` | Создает приглашение друга. |
| `GET` | `/leaderboard` | Возвращает топ пользователей по опыту орбиты. |
| `GET` | `/admin/kpi` | Возвращает текущие продуктовые и финансовые KPI. |
| `POST` | `/admin/simulate` | Создает демо-событие из админского симулятора. |
| `GET` | `/admin/risk` | Возвращает активные риск-флаги и награды в ожидании. |
| `GET` | `/admin/stream` | Отдает SSE-поток с обновлением KPI. |

## Мини-игры и награды

Каждая игра связана со своей планетой:

- `halva_snake` -> `ORBIT_COMMERCE`.
- `credit_shield_reactor` -> `CREDIT_SHIELD`.
- `social_ring_signal` -> `SOCIAL_RING`.

Если игра отправлена не на свою планету, API вернет ошибку. Это защищает баланс наград и не дает начислить звездную пыль в чужую ветку прогресса.

## Данные и сиды

При старте API создается демо-пользователь `u_demo`, базовые планеты, профиль рассрочки и стандартные квесты. Сиды обновляют названия и описания существующих квестов, поэтому русские тексты подтягиваются даже после предыдущих локальных запусков.

## Контракты

Команда `npm run contracts:generate` экспортирует `apps/api/openapi.json` и генерирует типы в `packages/contracts/src/generated/api.ts`. После изменения схем FastAPI обязательно обновляйте контракты перед сборкой фронтенда.

## Опциональная инфраструктура

```bash
docker compose up -d
```

Команда поднимет PostgreSQL и Redis. Для быстрого локального демо это не обязательно: API умеет работать с SQLite.
